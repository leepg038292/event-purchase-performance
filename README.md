## 📦 Event Purchase Performance

| 한림대학교 멋쟁이사자처럼 14기 백엔드 심화 세션

> 대규모 트래픽 환경에서의 이커머스 시스템 아키텍처 개선 및 성능 최적화 프로젝트
> 단순한 기능 구현을 넘어, 단일 서버에서 분산 시스템으로 진화하며 발생하는 병목 현상을 해결하고 아키텍처의 확장성(Scalability)을 실험합니다.

**참고도서**
- [가상 면접 사례로 배우는 대규모 시스템 설계 기초](http://www.yes24.com/Product/Goods/102819435)


### ⏱️ 프로젝트 개요
- 본 프로젝트는 단순한 기능 구현을 넘어, **아키택처의 단계적 진화**를 통해 성능을 검증하는 것을 목표로 합니다.
- 초기 단일 서버(V1) 에서 시작하여, 캐싱, 로드밸런싱, 서비스 분리를 적용하며 고성능 / 고가용성 구조 (V4) 까지 점진적으로 발전시킵니다.

### 🎯 프로젝트 목표
- 기능 구현 + 아키택처 설계 동시 진행
- 대량 트래픽 상황에서의 병목 분석 및 개선
- Redis, Load Balancing, 서비스 분리 적용
- 성능 비교 기반 구조
